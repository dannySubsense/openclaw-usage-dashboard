## 1. Overview

This specification defines the complete implementation for a local HTML dashboard that displays API token usage costs across Kimi (Moonshot) and Anthropic providers. The dashboard is generated by a single Python script that parses session JSONL files from OpenClaw, fetches live balance data from the Moonshot API, and produces a static HTML file with interactive Chart.js visualizations.

**Key Purpose**: Provide visibility into token usage patterns, costs, and anomalies to help manage API budgets effectively.

**Key Constraints**:
- Single Python file implementation
- No external dependencies beyond stdlib + requests
- Static HTML output only
- Chart.js via CDN only
- Must handle missing Moonshot API key gracefully

## 2. Invariants (Non-Negotiable)

The following rules MUST be maintained throughout implementation:

### 2.1 Architecture Invariants
1. **Single File**: All Python code must reside in `generate_dashboard.py` - no modules, no packages
2. **Minimal Dependencies**: Only `requests` and `python-dotenv` beyond Python stdlib
3. **Static Output**: Generated HTML must be self-contained and viewable offline (except Chart.js CDN)
4. **No Database**: Parse JSONL files directly on each run - no SQLite, no caching layer
5. **Chart.js CDN Only**: Use `https://cdn.jsdelivr.net/npm/chart.js@4.4.0/dist/chart.umd.min.js` - no local bundling

### 2.2 Data Integrity Invariants
1. **Cost Accuracy**: All costs must be stored and displayed in USD with 2 decimal precision
2. **Timestamp Preservation**: All timestamps must be ISO 8601 format, preserved from source
3. **Session Type Detection**: Cron sessions detected by `[cron:` prefix in message content - no other heuristics
4. **File Filtering**: Exclude files matching `*.deleted.*` or `*.reset.*` patterns
5. **Anomaly Definition**: Flag ANY Sonnet (`claude-sonnet-*`) or Opus (`claude-opus-*`) usage as fallback anomaly

### 2.3 Error Handling Invariants
1. **Graceful Degradation**: Missing Moonshot API key must NOT prevent dashboard generation
2. **Partial Data Acceptance**: Malformed JSONL lines must be skipped with warning, not abort processing
3. **API Failure Tolerance**: Moonshot API failures must display "N/A" in balance card, not crash
4. **Empty Data Handling**: Zero sessions found must generate empty dashboard with explanation, not error
5. **Exit Codes**: Only exit with code 1 for critical failures (missing session directory), use code 0 for all other scenarios

### 2.4 Security Invariants
1. **No Hardcoded Secrets**: API keys MUST come from environment only
2. **HTML Escaping**: All dynamic content (session IDs, timestamps) must be HTML-escaped
3. **Path Validation**: Session directory path must be validated before file access
4. **Permission Check**: .env file permissions must be checked, warn if world-readable

### 2.5 Output Invariants
1. **HTML5 Compliance**: Generated HTML must be valid HTML5
2. **Responsive Design**: Dashboard must work on mobile (< 768px), tablet (768-1024px), desktop (> 1024px)
3. **Color Consistency**: Use defined color palette (Moonshot: #4c8bf5, Anthropic: #d97706, etc.)
4. **Chart Data Format**: All chart data must be embedded in `<script>` tags, no external JSON files

---

## 3. Halt Conditions

Implementation MUST stop and raise error in these cases:

### 3.1 Fatal Errors (Exit Code 1)
1. **Session Directory Missing**: `config.session_dir` does not exist or is inaccessible
2. **Output Path Invalid**: Cannot write to specified output file path (permission denied, disk full)
3. **Invalid Python Version**: Python < 3.8 detected (dataclasses required)

### 3.2 Non-Fatal Errors (Log Warning, Continue)
1. **Moonshot API Failure**: HTTP error, timeout, or invalid response format
2. **Malformed JSONL Line**: JSON decode error on specific line
3. **Missing Usage Fields**: Message event lacks required `usage` or `cost` fields
4. **Invalid Timestamp**: Cannot parse timestamp string
5. **Empty Dataset**: No valid sessions found in time range

### 3.3 Configuration Errors (Use Default, Warn)
1. **Missing .env File**: Use empty string for API key, defaults for other config
2. **Invalid Budget Values**: Use defaults if non-numeric values provided
3. **Invalid Exchange Rate**: Use 0.14 default if invalid

#### 5.1.4 Test Cases
1. Valid JSONL with mixed message/session events → extracts only messages
2. File with malformed JSON line → skips line, logs warning, continues
3. Message with `content: "[cron: daily_backup]"` → session_type="cron"
4. Message with missing `usage.cost.total` → skips message
5. File named `session.deleted.jsonl` → file not processed
6. Empty JSONL file → returns empty SessionData list

#### 5.1.5 Implementation Notes
- Use `json.loads()` per line, catch `JSONDecodeError`
- Session ID: extract from filename (strip `.jsonl`)
- Start timestamp: first event timestamp
- End timestamp: last event timestamp
- Aggregate costs by summing `usage.cost.total` across all messages

    parser.add_argument('-o', '--output', default=os.getenv('OUTPUT_FILE', 'dashboard.html'))

## 10. API Reference

### 10.1 Moonshot Balance API
**Endpoint**: `https://api.moonshot.ai/v1/users/me/balance`
**Method**: GET
**Authentication**: Bearer token (API key)
**Headers**:
```
Authorization: Bearer {MOONSHOT_API_KEY}
```

**Success Response** (200):
```json
{
    "available_balance": 1234.56,
    "voucher_balance": 0.00,
    "cash_balance": 1234.56
}
```

**Response Fields**:
- `available_balance`: Total available balance (CNY) - sum of voucher and cash balances
- `voucher_balance`: Balance from vouchers/credits (CNY)
- `cash_balance`: Cash balance (CNY)

**Error Responses**:

**401 Unauthorized** - Invalid API key:
```json
{
    "error": {
        "message": "Invalid authentication credentials",
        "type": "invalid_request_error",
        "code": "invalid_api_key"
    }
}
```

**500 Internal Server Error** - Server error:
```json
{
    "error": {
        "message": "An internal server error occurred",
        "type": "api_error",
        "code": "internal_error"
    }
}
```

**Timeout** - Network timeout (no response body):
- Connection timeout after 10 seconds
- Handle with `requests.exceptions.Timeout`
- Display "N/A" in dashboard

**Example Request**:
```bash
curl -X GET https://api.moonshot.ai/v1/users/me/balance \
  -H "Authorization: Bearer $MOONSHOT_API_KEY"
```

**Handling**:
```python
try:
    response = requests.get(
        'https://api.moonshot.ai/v1/users/me/balance',
        headers={'Authorization': f'Bearer {api_key}'},
        timeout=10
    )
    response.raise_for_status()
    data = response.json()
    balance_cny = data['available_balance']
    return balance_cny
except requests.RequestException as e:
    logging.error(f"Moonshot API error: {e}")
    return None
```

## 12. Performance Considerations

### 12.1 File Processing
- Read JSONL line-by-line (generator pattern) to handle large files
- Filter files by date before parsing (if possible from filename)
- Stop parsing old files outside time range

### 12.2 Memory Usage
- Don't load all files into memory at once
- Use generators for file iteration
- Aggregate data incrementally

### 12.3 API Calls
- Single Moonshot API call per run
- 10-second timeout to prevent hanging
- No retry logic (fail fast)

### 12.4 HTML Generation
- Build HTML string in parts (use list + join)
- Avoid repeated string concatenation
- Pre-compute all chart data before template rendering
